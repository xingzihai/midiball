# 小球控制优化任务规划

## 问题诊断
当前 PlayerController 使用**加速度模型**（`velocity += input * accel * dt`），存在以下问题：
1. **输入延迟**：加速度模型需要时间积累速度，输入到响应有明显滞后
2. **滑行感**：damping=0.88 导致松手后球体持续滑动，难以精确停位
3. **无响应曲线**：线性映射，小幅度操作缺乏精度，大幅度操作不够快
4. **相机叠加延迟**：CameraFollow 的 xSmoothTime=0.15s 在控制延迟上再叠加视觉延迟

## 优化方案：直接速度映射 + 响应曲线

### 核心思路
参考专业音游/弹幕游戏的控制方案，采用**直接速度映射**替代加速度模型：
- 输入值直接映射为目标速度（零延迟响应）
- 幂次响应曲线：小输入=精确微调，大输入=快速移动
- 松手即停：输入归零时速度立即归零（可选极小平滑）
- 保留外部冲量接口（挡板反弹等）

### 修改清单
1. **PlayerController.cs** — 重写 `UpdateLateralMovement()`
   - 移除加速度模型，改为直接速度映射
   - 添加响应曲线（power=2.0）
   - 添加死区过滤（0.05）
   - 外部冲量独立衰减，不影响输入响应
   - 边界处理改为硬夹紧（移除反弹，反弹由挡板系统处理）

2. **CameraFollow.cs** — 降低 X轴平滑延迟
   - xSmoothTime 从 0.15 降至 0.06，减少视觉滞后

3. **GameConstants.cs** — 无需修改（常量仅作参考，实际参数在 SerializeField 中调节）

### 参数设计
| 参数 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| maxLateralSpeed | 50 | 35 | 直接映射不需要过高上限 |
| responseCurve | 无 | 2.0 | 幂次曲线，小输入精确 |
| deadZone | 无 | 0.05 | 过滤摇杆漂移 |
| impulseDecay | 无 | 8.0 | 外部冲量独立衰减速率 |
| xSmoothTime(相机) | 0.15 | 0.06 | 减少视觉延迟 |
